## Q1
**答え：B**

### ポイント
- 代入演算子 `=` は右辺を引数に取るため、呼び出しの際は右結合となる。
- `x = y = z = 15;` は `x = (y = (z = 15));` と解釈される。
- まず `z` に `15` が代入され、その結果（`z` への参照）が `y`、さらに `x` へ渡るため、最終的に3変数すべて `15` になる。

**誤答の理由**
- **A**: 左結合として読んでしまう誤り
- **C**: `=` は連鎖可能であり、これは正しい C++ コード
- **D**: `=` の結果型を `bool` と誤解しており、比較演算子 `==` と混同している

## Q2
**答え：B**

### ポイント
- `(wallet += salary) += bonus;` が組み込み型と同じように連鎖するには、`operator+=` が更新後の左辺オブジェクトそのものを返す必要がある。
- 戻り値が `Yen`（値返し）だと、1 回目の `+=` の結果がコピーになり、2 回目はそのコピーに対して実行される。
- 戻り値を `Yen&` にすると、2 回目も同じ `wallet` 本体に作用する。

#### Yen& operator+=(...) の場合（正しい）

| 手順 | 実際に操作される対象 | 計算 | wallet の値 |
| --- | --- | --- | --- |
| 初期状態 | wallet | 150000 | 150000 |
| ① `wallet += salary` | wallet 本体 | 150000 + 250000 | 400000 |
| ② `return *this` | wallet 本体への参照 | 同じ wallet を返す | 400000 |
| ③ `+= bonus` | wallet 本体 | 400000 + 1000000 | 1400000 |

#### Yen operator+=(...) の場合（値返し）

| 手順 | 実際に操作される対象 | 計算 | wallet の値 |
| --- | --- | --- | --- |
| 初期状態 | wallet | 150000 | 150000 |
| ① `wallet += salary` | wallet 本体 | 150000 + 250000 | 400000 |
| ② `return *this` | wallet のコピー | コピー生成 | 400000 |
| ③ `+= bonus` | コピー | 400000 + 1000000 | wallet は変化なし |
| ④ コピー破棄 | コピー | 消滅 | 400000 |

**誤答の理由**
- **A**: 型拡張（`int` -> `long long`）は今回の本質ではない
- **C**: 引数を値渡しにしても、連鎖で問題になるのは戻り値側
- **D**: `const Yen&` だと `(wallet += salary)` が `const` 扱いになり、その後の `+= bonus`（非 `const` メンバ関数）が呼べない
